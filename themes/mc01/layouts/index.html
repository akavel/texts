Nix explorations ep.1: Declarative per-user configuration — 8.09.2017, 11 min.

nix nixos

One of the defining features of Nix is that it enables easy declarative configuration. In case of NixOS (the Nix-based Linux distribution), this specifically means that the whole OS is totally and deterministically configured by a single text file — located at: /etc/nixos/configuration.nix. As used with Nix, this gives multiple important benefits — one can take this file to another PC and easily build an exact1 clone of the original OS; tweaking configuration is safe because any changes can be easily reverted; at any moment, whole system configuration can be quickly backed up; and much more.

multigit, or How To Allow Multiple Git Repos To Coexist Peacefully in The Same Directory — 8.09.2017, 11 min.

nix utils

Some 5 years ago I was fairly annoyed by a small problem I couldn’t solve: I had two groups of scripts in my ~/bin/ directory at work (employer-specific vs. universal), and I wanted to keep them all versioned, but in two separate repositories (corresponding to the two groups). The employer-specific scripts were useless outside my office; they were project-specific, had hardcoded paths, etc. But I still wanted them backed up and versioned. The other group were some universal scripts. Those were "timeless", I copied them from a different machine before, and would copy them again to some other machine in future. They were general, and I wanted to share them easily (and versioned too). And I didn’t want to have them "contaminated" by the job-only ones. At the time I didn’t resolve the problem, although I’ve had some initial thoughts.
